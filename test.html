Author And Notes API using Express and MongoDB

Create Author

Endpoint: POST /users/signup

Format:

{
  "username": "Author name",
  "email": "Author email",
  "password": "PASS"
}

Login

Endpoint: POST /users/signin

Request: Attach authentication token in the Header section with key Authorization and value Bearer "token"

Create Note

Endpoint: POST /notes/

Request: Attach authentication token of logged-in user in the Header section with key Authorization and value Bearer "token"

Delete Note

Endpoint: DELETE /notes/Attach note ID

Request: Attach note ID

Update Note

Endpoint: PUT /notes/{noteID}

Get All Notes

Endpoint: GET /notes/

API Documentation on Postman

Testing API Endpoints

To test the API endpoints, you can use tools like Postman and Thunder Client in Visual Studio Code:

  • Postman: Download and install Postman. Create requests for each endpoint using the provided methods and paths.
  • Thunder Client (VS Code Extension): If you're using Visual Studio Code, you can install the Thunder Client extension. It's a lightweight REST client that you can use to send requests directly from VS Code.

Remember to attach the required headers or tokens as mentioned in the endpoint descriptions.
